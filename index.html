Projet : Modélisation et Intégration d'une Base de Données sur les Catastrophes Climatiques

Contexte : Projet réalisé dans le cadre de la SAÉ Base de Données (BUT1 Informatique)[cite: 4].

Objectif : Concevoir, déployer et peupler une base de données relationnelle permettant d'analyser la fréquence et l'impact des catastrophes climatiques à l'échelle mondiale[cite: 4].

Outils et Technologies : SQL (PostgreSQL, MySQL), MySQL Workbench (AGL), modélisation relationnelle, processus ETL[cite: 4].

Mes réalisations clés :
  • Conception et Modélisation : Création de l'architecture relationnelle de la base autour de 5 tables principales connectées[cite: 4].
  • Analyse comparative : Étude critique des différences d'implémentation (contraintes, encodage, portabilité) entre des scripts SQL produits manuellement et ceux générés automatiquement par l'AGL[cite: 4].
  • Pipeline ETL (Extract, Transform, Load) : Importation de données brutes depuis un fichier CSV vers une table de transition sous PostgreSQL via la commande \copy[cite: 4].
  • Nettoyage et Intégration : Peuplement sécurisé des tables finales avec plus de 6 400 enregistrements tout en garantissant l'intégrité référentielle et l'élimination des doublons via des requêtes INSERT INTO ... SELECT DISTINCT[cite: 4].

SAÉ S3 : Suivi d'une cohorte d'étudiants

Contexte : En équipe de 5, nous avions comme projet universitaire de réaliser une application web qui permet de suivre une cohorte d’étudiants[cite: 4].

Définition : Une application web destinée à agréger, gérer et visualiser les données de progression d'un groupe d'étudiants sur l'ensemble de leur cursus BUT[cite: 4].

Mon rôle et mes missions :
  • Chacun des membres du groupe avait une tâche spécifique à réaliser seul de son côté. Ma tâche principale durant cette SAE a été de coder l'architecture interne du site[cite: 4].
  • Backend & Modèle MVC : J'ai utilisé le framework Python Flask, en structurant l'application selon le modèle MVC (Modèle-Vue-Contrôleur). Je me suis occupé de toute la partie Modèle, avec le codage de toutes les interfaces et tous les DAO[cite: 4].
  • Lien BDD-Site : Cette partie est sans doute l’une des plus importantes, car c’est dans celle-ci qu’on fait le lien direct entre la base de données et l'interface du site[cite: 4].
  • Compétences acquises : Le codage de cette partie m’a permis de comprendre l'architecture logicielle globale, d'apprendre à manipuler rigoureusement le modèle MVC, et de renforcer mes compétences en programmation orientée objet avec Python. Une fois mon travail terminé, je l'ai présenté et expliqué au reste de l'équipe[cite: 4].
Base de données :
  • Une autre partie dans laquelle je suis intervenu est la conception et l'écriture du script de la base de données. J'y ai acquis des compétences techniques solides en SQL, ainsi que des compétences transversales (travail d'équipe, synchronisation des données)[cite: 4].
